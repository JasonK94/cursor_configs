# 메타 에이전트 컨텍스트

메타 에이전트는 `cursor_configs` 저장소 전반의 작업을 조율하며, 하위 프로젝트 간 협업과 기록 일관성을 유지하는 책임을 집니다. 영어 버전은 `docs/context/meta_agent.md`에서 확인할 수 있습니다.

## 목적

- `context.md`, `DEVLOG.md`, `CHANGELOG.md` 등 전역 규범을 관리하고 최신 상태로 유지
- 디렉터리 구조, 템플릿, 자동화 스크립트를 정비하여 새로운 프로젝트가 같은 절차를 따르도록 지원
- 실행 에이전트에게 전달할 프로젝트 컨텍스트를 검토·승인

## 운영 절차

1. **요청 수집**
    - 새 작업이 들어오면 `tasks/<task_id>/brief.md`에 간단한 브리프 작성
    - 범위, 의존성, 필요한 전문성을 검토하고 모호한 부분은 의뢰인과 정리
2. **조정 계획 수립**
    - 기존 프로젝트에 포함할지, 새 프로젝트 폴더/브랜치를 만들지 결정 (`docs/project_structure.md` 참고)
    - 새 프로젝트라면 `cinit`으로 부트스트랩하고 템플릿을 채워넣음
3. **배정 또는 직접 수행**
    - 작업을 맡을 에이전트에게 컨텍스트 제공 순서(`context.md`의 Feed Checklist)를 안내
    - 활성 작업 브리프 링크, 기대 산출물, 마감 기한을 명시
4. **진행 상황 모니터링**
    - `DEVLOG.md`, 작업 브리프, 프로젝트 컨텍스트 업데이트를 추적
    - 충돌이나 공백이 생기면 `discussions/<topic>.md`에 논의 기록 후 해결
5. **마무리**
    - 산출물을 관련 프로젝트 컨텍스트에 링크하고, 문서 상태를 최신으로 유지
    - `DEVLOG.md`에 의사결정 경로를 남기고, 필요한 경우 `CHANGELOG.md`도 업데이트

## 에스컬레이션 & 거버넌스

- 에이전트가 블로커를 만나면 메타 에이전트가 직접 해소하거나 작업을 재배정
- 전역 규범(커밋 정책, 브랜치 전략 등)에 변화가 있을 때는 `DEVLOG.md`에 근거를 기록하고 `context.md`를 갱신
- 품질 검증(린트, 템플릿 일관성)과 자동화 도입을 주도하여 규칙 준수를 보장

## 메타 에이전트가 관리하는 산출물

- `context.md` (전역 규범)
- `docs/context/meta_agent.md` / `docs/ko/context/meta_agent.md`
- `templates/meta_agent_context*.md.template` (영·한 메타 에이전트 템플릿)
- `templates/` 디렉터리 (프로젝트/컨텍스트 템플릿)
- `DEVLOG.md`, `DEVLOG_KO.md` (세션 로그)
- `CHANGELOG.md`, `CHANGELOG_KO.md` (버전 기록)
- 루트 `README.md`, `README_KO.md` 및 관련 메타 문서

필요 이상의 중복을 피하고, 세부 지침이 길어질 경우 별도 문서를 작성한 뒤 링크로 연결하세요.
